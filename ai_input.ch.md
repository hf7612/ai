# 输入法状态听觉感知自动提示方案

## 项目简介
日常使用电脑时，我们经常需要中英文混合输入，很多时候无暇注视屏幕，稍有分心就无法分辨当前处于中文还是英文输入法状态，极易出现输错字符、反复删除修改的情况，严重影响输入体验与效率。

基于日常使用痛点，本项目实现一套**后台无感监听 + 听觉状态区分**的轻量化解决方案：
切换至中文输入法时关闭按键音效保持静音，切换至英文输入法时自动开启复古按键音效，仅凭声音即可快速判断当前输入法状态，实现盲打精准区分。

## 实现原理
1. **获取键盘设备节点**
通过 Linux 系统路径 `/sys/class/input/event*` 遍历筛选本机对应的键盘输入设备节点。

2. **底层事件阻塞监听**
打开 `/dev/input/event*` 输入设备文件，采用阻塞方式实时读取系统原生 `input_event` 键盘事件。

3. **精准捕获输入法切换动作**
通过三重条件判定输入法切换快捷键触发行为：
type == EV_KEY && ev.code == KEY_WAKEUP && ev.value == 1
满足条件即判定用户完成输入法切换，自动调用状态检测脚本。

4. **输入法状态判定与进程管理**
调用 `fcitx5-remote` 获取实时输入法状态，循环校验状态稳定性，根据语种自动启停 `buckle` 按键音效进程：
- 中文输入法：终止音效进程，输入全程静音
- 英文输入法：后台启动音效进程，敲击键盘播放提示音

## 核心执行脚本
```bash
#!/bin/bash
# 输入法状态监听与音效启停脚本
for x in `seq 5`;do
    i=`fcitx5-remote`
    if [ "y""$i" == "y1" ];then
        # 中文输入法，关闭按键音效
        killall -9 buckle
    else
        # 英文输入法，开启按键音效
        if [ -z $(pidof buckle) ];then
            buckle &
        fi
    fi
    sleep 0.2
done
整体运行流程:
后台常驻监听程序，静默抓取系统键盘底层事件
触发输入法切换快捷键，程序识别切换行为
脚本多次校验稳定输入法运行状态
自动匹配启停按键音效程序
依靠听觉区分输入状态，无需视线查看屏幕

项目优势:
极低资源占用，后台静默运行不影响日常使用
脱离屏幕依赖，分心操作、盲打场景均可适用
基于 Linux 原生输入接口开发，兼容性强运行稳定
代码逻辑简洁，易于自定义修改、拓展更多功能
纯本地实现，无网络依赖，安全无额外权限风险
